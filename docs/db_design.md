# 도서 대여 관리 시스템 - 상세 설계 명세서 (Database Design Specification)

본 문서는 도서 대여 관리 시스템의 데이터베이스 스키마 설계 기준, 제약조건 설정 근거 및 핵심 SQL 쿼리 17개의 상세 요구사항 분석 내용을 보관하는 상세 명세서입니다.

---

## **1. 컬럼 데이터 타입 선정 기준 및 이유**

시스템의 저장 효율성과 인덱스/조인 연산 속도를 보장하기 위해 다음과 같은 설계 기준으로 컬럼 타입을 선정하였습니다.

| 데이터 타입 | 적용 컬럼 | 선정 이유 및 특징 |
| :--- | :--- | :--- |
| **`INT`** | `id`, `price`, `category_id`, `member_id`, `book_id` | - **조인 및 검색 속도 향상**: 테이블 간 조인(JOIN) 및 관계 매칭의 기준이 되는 PK/FK 식별자 컬럼들은 CPU 연산과 비교 속도가 가장 빠른 기본 정수형(`INT`)을 채택했습니다.<br>- **데이터 특성 반영**: 도서 가격(`price`)은 소수점이 필요하지 않고 국내 원화 정수형 범위 내에 위치하므로 메모리 낭비를 유발하는 실수형 대신 `INT`를 지정했습니다. |
| **`VARCHAR(N)`** | `email`, `name`, `title`, `author` | - **가변 데이터 크기 최적화**: 이메일, 회원명, 도서명 등은 고정 문자 타입(`CHAR`) 사용 시 미사용 공간이 낭비되므로, 실제 저장되는 글자 수만큼의 바이트만 차지하는 가변 길이 타입인 `VARCHAR`를 적용했습니다.<br>- **보안 및 규격 준수**: 이메일 규격(최대 100자), 한글/영문 이름의 일반적 길이(최대 50자), 긴 도서명(최대 255자) 등 비즈니스 요구 규격을 상한선으로 두어 공간 낭비를 막고 데이터 유실을 방지했습니다. |
| **`DATETIME`** | `created_at`, `rented_at`, `returned_at` | - **초 단위의 이력 추적**: 도서 대여/반납 및 회원 가입은 단순 날짜(`DATE`)뿐만 아니라 시/분/초 단위까지 정밀하게 추적되어야 하므로 날짜와 시간 정보가 결합된 타입을 선정했습니다.<br>- **안정성**: DB 서버 시간대(Timezone) 설정 변경의 영향을 덜 받아 로컬 날짜와 시간을 일관성 있게 유지하고, 표현 범위가 훨씬 넓은(1000-01-01 ~ 9999-12-31) `DATETIME`을 사용하여 데이터 정합성을 유지했습니다. |

> **SQL 데이터 타입(Data Type) 요약**
> * **`INT`**: 소수점이 없는 **정수형** 데이터를 저장하는 타입입니다. (예: 가격, 회원 ID, 카테고리 ID 등)
> * **`VARCHAR(N)`**: 입력된 텍스트 크기에 맞춰 가변적으로 공간을 차지하는 **가변 길이 문자열** 타입입니다. `N`은 입력 가능한 최대 글자 수 제한을 의미합니다. (예: 이메일, 이름, 책 제목 등)
> * **`DATETIME`**: `YYYY-MM-DD HH:MM:SS` 형태로 **날짜와 시간**을 초 단위까지 정확하게 저장하는 시간 타입입니다. (예: 회원 가입 일시, 대여 일시 등)

## **2. 제약조건 적용 현황 및 비즈니스 규칙**

데이터의 무결성과 신뢰성을 확보하기 위해 SQL 표준 제약조건을 아래와 같이 적용했습니다.

| 적용 테이블 | 컬럼명 | 제약조건 유형 | 적용 이유 및 비즈니스 규칙 |
| :--- | :--- | :--- | :--- |
| **모든 테이블** | `id` | **PRIMARY KEY** | 각 행(Row)을 유일하게 식별하고 조회 성능을 높이기 위해 기본키 지정 |
| `member` | `email` | **UNIQUE** | 한 명의 회원은 하나의 이메일만 가질 수 있으며, 중복 가입을 방지함 |
| `member` | `email`, `name` | **NOT NULL** | 회원 가입 및 관리 시 필수적으로 채워져야 하는 필수 정보로 지정 |
| `category` | `name` | **NOT NULL** | 카테고리 이름 없이 분류가 생성되는 것을 방지함 |
| `book` | `title` | **NOT NULL** | 도서 등록 시 제목이 누락되는 데이터 오류를 방지함 |
| `rental` | `member_id` | **FOREIGN KEY** | `member` 테이블의 `id`를 참조하며, 존재하지 않는 회원의 유령 대여 입력을 차단함 (참조 무결성) |
| `rental` | `book_id` | **FOREIGN KEY** | `book` 테이블의 `id`를 참조하며, 시스템에 등록되지 않은 도서의 잘못된 대여 입력을 차단함 (참조 무결성) |
| `rental` | `rented_at` | **NOT NULL** | 도서가 대여된 시점의 날짜/시간 데이터는 누락될 수 없도록 강제함 |

## **3. 핵심 SQL 쿼리 상세 분석 (총 17개)**

각 쿼리의 구체적인 SQL 코드는 [queries.sql](../queries.sql)에서 확인할 수 있으며, 실행 예시는 [results/](../results/) 폴더의 개별 파일들로 확인할 수 있습니다.

### 3.1. 기본 조회 쿼리 (최소 4개 이상)
* **Q1. 고가 도서 목록 조회**: 가격이 20,000원 이상인 도서 목록을 가격이 높은 순으로 정렬하여 조회합니다. (`WHERE` 필터링 및 `ORDER BY DESC` 활용)
* **Q2. 특정 성씨 회원 검색**: 성이 '김'씨인 회원 목록을 `LIKE` 연산자를 사용하여 조회합니다. (`WHERE ... LIKE` 조건 검색)
* **Q3. 최신 가입 회원 조회**: 회원 가입일 역순으로 정렬하여 최근 가입한 회원 3명만 조회합니다. (MySQL 고유의 `LIMIT` 문법 사용 주석 설명 포함)
* **Q4. 대여 중인 기록 조회**: 반납일(`returned_at`)이 `NULL`인 미반납 대여 기록들을 대여일 기준 오름차순으로 조회합니다.

### 3.2. 조인(JOIN) 조회 쿼리 (최소 4개 이상)
* **Q5. 대여 중인 도서 및 회원 정보 조회 (`INNER JOIN`)**: 대여 기록과 회원, 도서 테이블을 결합하여 현재 미반납 대여 건의 도서명과 대여한 회원 이름을 조회합니다.
* **Q6. 도서별 카테고리 매칭 조회 (`INNER JOIN`)**: 도서 테이블과 카테고리 테이블을 연결하여 도서의 제목과 카테고리명을 결합해 조회합니다.
* **Q7. 전체 회원의 총 대여 횟수 조회 (`LEFT JOIN`)**: 대여 기록이 없는 회원까지 모두 포함하여 각 회원의 고유 ID, 이름 및 총 대여 횟수를 집계합니다.
* **Q8. 특정 카테고리 도서 조회 (`INNER JOIN` + `WHERE`)**: 'IT/컴퓨터' 카테고리에 속한 도서들의 제목과 저자명을 조인하여 조회합니다.

> [!NOTE]
> **INNER JOIN vs LEFT JOIN 비교**
> 두 조인(JOIN) 방식의 작동 원리와 문법적 차이에 대한 상세한 설명은 [SQL 쿼리 활용 가이드 - JOIN 편](query_guide.md#c-테이블-결합-join)에서 확인할 수 있습니다.

### 3.3. 집계 및 그룹화 쿼리 (최소 3개 이상)
* **Q9. 카테고리별 도서 수량 및 평균 가격 집계**: 각 카테고리 ID별로 속한 도서의 총 수량과 평균 가격을 그룹화하여 산출합니다. (`GROUP BY` + `COUNT` + `AVG`)
* **Q10. 회원별 반납 완료 대여 건수 집계**: 회원 ID별로 이미 반납 처리가 완료된(`returned_at IS NOT NULL`) 대여 건수를 집계합니다. (`WHERE` 필터링 후 `GROUP BY`)
* **Q11. 우수 회원 ID 추출 (`HAVING` 활용)**: 누적 대여 건수가 2건 이상인 회원의 ID와 대여 횟수를 그룹화하여 조건 필터링합니다. (`GROUP BY` + `HAVING`)
* **Q12. 카테고리별 도서 총 가격 합계(SUM) 및 평균 가격(AVG) 조회**: 도서 테이블과 카테고리 테이블을 조인하여 각 카테고리명 별로 도서 가격의 합계와 평균을 구합니다. (`JOIN` + `GROUP BY` + `SUM` + `AVG`)

### 3.4. 서브쿼리(Subquery) 쿼리 (최소 1개 이상)
* **Q13. 대여 이력이 없는 회원 조회 (`NOT IN` 서브쿼리)**: 대여 테이블의 회원 ID 목록에 없는 회원들의 이름을 조회합니다.
* **Q14. 평균 가격보다 비싼 도서 조회 (비교 서브쿼리)**: 서브쿼리를 이용해 전체 도서의 평균 가격을 구한 뒤, 그보다 가격이 높은 도서들의 제목과 가격을 출력합니다.

### 3.5. 데이터 수정 및 삭제 쿼리 (최소 2개 이상)
* **Q15. 도서 반납 처리 업데이트 (`UPDATE`)**: 특정 회원(1번)이 빌린 특정 도서(2번)의 반납일(`returned_at`)을 현재 시각으로 수정합니다.
* **Q16. 유령 도서 데이터 일괄 삭제 (`DELETE`)**: 카테고리 ID가 누락되어 연결이 끊어진 도서 정보를 테이블에서 삭제합니다.

### 3.6. 인덱스 정의 (최소 1개 이상)
* **Q17. 대여일 검색 성능 향상을 위한 인덱스 생성 (`CREATE INDEX`)**
    * **적용 구문**: `CREATE INDEX idx_rental_rented_at ON rental(rented_at);`
    * **적용 이유**: 대여일(`rented_at`) 컬럼이 대여 현황 조회나 정렬 등 `WHERE` 및 `ORDER BY` 절에 빈번하게 사용되므로, 검색 속도를 향상시키기 위해 적용했습니다.
